# 测试需求文档 - 概述

## 1. 文档目的

本文档描述乐高故事书籍系统的测试需求，包括单元测试、集成测试、端到端测试的测试用例。

## 2. 测试范围

### 2.1 功能测试范围

| 子系统 | 测试内容 |
|--------|----------|
| 用户管理 | 登录、登出、用户信息管理 |
| 人仔管理 | 预设人仔、自定义人仔CRUD |
| 角色管理 | 角色分配、自定义名称、角色类型 |
| 情节管理 | 预设情节、自定义情节、语音输入 |
| 故事生成 | AI故事生成、关键词高亮 |
| 解密互动 | 谜题验证、答题记录、提示机制 |
| 章节管理 | 章节CRUD、章节阅读 |
| 书籍管理 | 书籍CRUD、回收站 |
| 分享管理 | 分享创建、访问控制 |
| 家长控制 | 时长控制、内容过滤 |

### 2.2 非功能测试范围

| 类型 | 测试内容 |
|------|----------|
| 性能测试 | 页面加载时间、API响应时间 |
| 安全测试 | 认证授权、数据隔离 |
| 兼容性测试 | 浏览器兼容、移动端适配 |

## 3. 测试类型

### 3.1 单元测试
- 测试对象：单个函数、模块
- 测试工具：Jest
- 覆盖率要求：每个函数都要包括，代码各个分支都要覆盖
- 数据库：连接真实的Cloudflare D1数据库
- AI功能：使用mock API

### 3.2 集成测试
- 测试对象：模块间交互、API接口
- 测试工具：Jest + Supertest
- 数据库：连接真实的Cloudflare D1数据库
- AI功能：使用mock API

### 3.3 端到端测试
- 测试对象：完整用户流程
- 测试工具：Playwright
- 数据库：连接真实的Cloudflare D1数据库
- AI功能：使用mock API

## 4. 测试环境

### 4.1 环境配置
- Node.js: v18+
- Cloudflare D1: 真实数据库
- Cloudflare R2: 真实存储（测试用base64模拟）
- API密钥：通过环境变量配置

### 4.2 测试数据
- 预设人仔数据
- 测试用户数据
- 测试书籍数据
- 测试章节数据

## 6. 测试原则

### 6.1 测试独立性
- 每个测试用例独立运行
- 测试前清理数据
- 测试后恢复环境

### 6.2 测试可重复性
- 相同输入产生相同输出
- 使用固定的测试数据
- 避免依赖外部状态

### 6.3 测试覆盖率
- 正常流程覆盖
- 异常流程覆盖
- 边界条件覆盖

## 7. Mock策略

### 7.1 AI功能Mock
- 语音识别API：返回固定文本
- 图生图API：返回固定图片URL
- 文本生成API：返回固定故事内容

### 7.2 外部服务Mock
- 不依赖外部服务的实时状态
- 使用预定义的响应数据

## 8. 测试执行流程

```
1. 准备测试环境
   ↓
2. 初始化测试数据
   ↓
3. 执行单元测试
   ↓
4. 执行集成测试
   ↓
5. 执行端到端测试
   ↓
6. 生成测试报告
   ↓
7. 清理测试数据
```

## 9. 验收标准

### 9.1 功能验收
- 所有测试用例通过
- 覆盖率达到要求
- 无严重缺陷

### 9.2 性能验收
- API响应时间 < 3秒
- 页面加载时间 < 5秒

### 9.3 安全验收
- 无安全漏洞
- 数据隔离正确
